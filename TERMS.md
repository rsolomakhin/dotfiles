# Terms Used in This Repository

This file defines terms and acronyms used in this repository.

-   Aliases: Shortcut names for commands in Bash.
-   Auto-completion: Suggesting completions for text as you type.
-   `coc.nvim`: "Conquer of Completion", a Vim plugin for auto-completion and
    Language Server Protocol (LSP) support.
-   `common.sh`: A script sourced by both Bash and Zsh to share configuration.
-   Cursor Shape: The visual appearance of the cursor (block or pipe).
-   CVS: Concurrent Versions System.
-   Definition: The location where a symbol (function, variable) is defined.
-   Diagnostic: Error or warning messages shown by the editor (via `coc.nvim`).
-   Dotfiles: Configuration files for the user environment, typically starting
    with a dot (e.g., `.vimrc`, `.bashrc`).
-   `fzf`: Fuzzy Finder, a tool used for searching files and history.
-   `.gitconfig.user`: A local file used to store Git user identity (name and
    email), separate from the tracked `gitconfig`.
-   Home Directory (`$HOME` or `%USERPROFILE%`): The target location where
    configuration files are linked.
-   Leader Key: A special key in Vim used to trigger custom mappings when in
    normal or visual mode. In this config, it is `\`.
-   LSP: Language Server Protocol, a protocol used to provide language features
    like auto-complete, go to definition, etc.
-   Modes (Vim): Normal, Visual, and Insert modes.
-   Prompts: The text shown by the shell waiting for input.
-   References: Places where a symbol is used.
-   Symlink / Symbolic Link: A file that points to another file or directory.
    Used to link files from the repository to the user's home directory.
-   `vim-plug`: The plugin manager used for Vim.
-   Vim: "Vi IMproved", a text editor configured by these dotfiles.
