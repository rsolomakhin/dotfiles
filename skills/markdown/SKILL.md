# Markdown Files Skill

This file defines the skill for managing and improving Markdown files.

## Instructions

When modifying Markdown files:

-   Format markdown files to 80 characters per line.
-   Never put full paths in this repo. All paths must be relative to `$HOME` or
    the root of this repository.
-   Lists must start with a newline (leave a blank line before the list).
-   Section and subsection titles must be followed by a newline.
-   Files and code symbols should be formatted with backticks (`).
-   Bullet points should use `-`.
-   Bold text is not preferable.
